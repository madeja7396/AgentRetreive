# SIer_SOUL Charter

更新日: 2026-02-28

## 定義

SIer_SOUL は、AgentRetrieve の保守運用で守るべき「行動規範 + 実務規律」の総称。
スローガンではなく、実行可能な規則として運用する。

## 8つの原則

1. 契約優先: 実装より先に契約を確認する
2. 証跡優先: 口頭報告ではなく再実行可能な証跡を残す
3. 最小変更: 目的に対して必要最小限の差分で解決する
4. 再現性: 同じ入力で同じ結果を再現できる状態を維持する
5. 停止判断: 異常を検知したら進めず止める
6. 分離統治: 探索結果と公式結果を混同しない
7. 引き継ぎ可能性: 次担当が当日中に追従できる情報を残す
8. 継続改善: 失敗を lessons に変換して再発率を下げる

## 行動規範（必須）

- 変更時は必ず検証コマンドを実行する
- 結果を `tasks/todo.md` レビュー欄に記録する
- 修正学習を `tasks/lessons.md` に追記する
- 作業完了時に未完了項目を明示する

## 禁止規範

- 推測で評価値を更新する
- 失敗ログを残さず再試行を繰り返す
- 「誰かが知っているはず」を前提に運用する
- 一時対応を恒久運用として放置する

## 受け入れ基準

- スタッフエンジニアが「この変更は監査可能」と判断できること
- 新規担当が docs と tasks だけで再実行できること
- 主要運用導線（validate / experiment-ready / report）が破綻していないこと
