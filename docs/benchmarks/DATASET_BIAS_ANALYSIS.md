### 2. 層化サンプリングの欠如
現状のタスクはドキュメントファイルに集中。ソースコード（src/）やテスト（tests/）への検索が評価されていない。

## 緩和策（将来対応）

### Phase A: ドキュメント内分散（v1.2）
- CHANGELOG.md, CONTRIBUTING.md, docs/*.mdへの分散
- 目標: README.md比率を50%以下に

### Phase B: ソースコード検索（v2.0）
- シンボル定義検索タスク（関数定義、型定義）
- エラーメッセージ検索タスク
- 設定パラメータ検索タスク

### Phase C: クロスカテゴリ（v2.1）
- 「このエラーの発生箇所」（tests/ → src/）
- 「この設定のデフォルト値」（docs/ → src/）
